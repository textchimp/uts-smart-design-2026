What is Coding?

  • Writing step-by-step instructions in a way a computer can understand in order to solve a problem... kind of like a recipe
  • ...i.e., being so clear and specific about what you want done, that even a dumb literal computer can do it for you
  • ...using a programming language, which is halfway between:
    • vague-but-intuitive human languages, and
    • the precise-but-difficult language of mathematics
  • We'll be using the Arduino language - a kind of customised C
  • Programs are all a matter of input → processing → output
  • You may have already done programming without knowing it?
  • Warning! Get comfortable with "Error-Driven Development":
    • 30% - writing new lines of code
    • 70% - figuring out why they don't work (usually typos)

Data Types

"What kinds of information can I work with?"

  • Numbers:
    • int (integers/whole numbers): 1, 73542, -11, 0 ...
    • unsigned long for very long positive integers like timestamps
    • float (floating point numbers/decimals/fractions): 1.0, 0.5, 3.14159, -84632.1
  • char (single characters): 'a', 'A', 'x', '&' ...
  • String (longer strings of characters): "Hello World", "Please enter password:"
  • bool: (boolean true/false, yes/no answers): only true and false
  • A list of any of the above types ("array"): []
  • Lots of custom types when you start using libraries (classes/OOP)

Variables

  • A variable is a container for storing data
  • It lets us give a clear, unique name to a piece of information
  • It has two parts:
    • its name ("identifier") - make them as descriptive as possible!
    • the value being stored
  • To declare and initialise a variable, also specify its type: 
// Make me an integer var called `age` and store 25 into it 
int age = 25;

// Once created, you can change what's stored in it
age = 30;
age = age + 20;   // we can do maths

// You CAN'T change the TYPE of data you store in it
age = "Thirty";   // ‚ùå NO  - "invalid conversion" error

String ageInWords = "Thirty";    // ‚úÖ  OK (make a new variable)

Functions

  • A function is a named container for grouping related lines of code
  • One of the most important building blocks of code; most programs are just functions calling each other (divide and conquer)
  • They can (optionally) have inputs (arguments) and outputs (return values) - in this way they're like miniature programs
  • There are two aspects to a function:
    • defining the function (unless it's pre-defined)
    • using (a.k.a. running/calling/invoking) the function
  • Arduino comes with a lot of pre-defined functions ready to use. These are like handy little reusable pieces of code, which you use to communicate with the electronics components in your project

Functions II

  • Most of the time we'll run pre-defined or "built-in" functions that Arduino makes available to us:
  • Some take inputs (arguments), some don't; some give outputs that we can keep (return values), some don't. All combos are possible.
    • // Run the noInterrupts function
// Note there are no inputs or outputs; just does something "behind the scenes"
noInterrupts();

// Run the delay function and tell it how long to delay (no return value)
delay(1000); 

// Run the random function and save the number it returns into a variable
// ...give it two arguments which are the min and max range for the number
int diceRoll = random( 1, 7 );    // could also write e.g. random( 10 )

// Run the millis function and store its return value in a variable for later
// No inputs (argument), but important outputs (return value)
unsigned long runningTime = millis(); 

Serial.println( runningTime ); // use the variable 

// Enable serial communication (over USB) using this built-in function
// It lives inside a toolkit variable of related functions called Serial:
Serial.begin(115200);   // Give the speed as an argument

Functions III

  • In Every Arduino project ("sketch") we'll define two functions which Arduino expects to exist; they will automatically be run for us at the right time (we never run them "manually" ourselves): 
    // Put special `global` variables at the top of the file
int myCounter = 0;


// `setup` runs ONCE when the Arduino boots
void setup(){
  Serial.begin(115200);
  // Other one-off setup code here
} 

// `loop` runs repeatedly, forever, after setup is finished
void loop(){
  Serial.println( "Hello world." );
  myCounter = myCounter + 1;   // myCounter++;
  delay(10);
}
  • You might even define and run your own custom functions

Conditionals

  • Conditionals, a.k.a branching, a.k.a if-else statements, let our programs do different things at different times, depending on the answers to questions we ask about our data: 
    
if( buttonHeldTime > 1000 ){
  openDoor();
} else {
  Serial.println("Still closed...");
}
  • The expression inside the parentheses: buttonHeldTime > 1000 is the condition - the question we are asking; it always has a yes/no answer, i.e. a boolean true/false value.
  • You can ask lots of questions to compare data in different ways: ==, <, >, <=, >=, !=

Debugging

  • üéµ "If in doubt, print it out" üéµ: 
    int buttonTimer = 500;

// ... followed by some other code that maybe doesn't do what you expected...

Serial.print( "buttonTimer is: " );
Serial.println( buttonTimer );    // Will print  "Button timer: 500"
  • Notice and read any errors given! Check the line numbers
  • Style: indent correctly to make your code readable; use blank lines for clarity
  • Write comments to explain unclear code to yourself and others
  • Notice the font colours in your code editor! They are giving you information and often highlighting problems
  • Most common errors are typos: check spelling, case sensitivity, keywords
  • Next most common are syntax errors: match all parentheses and brackets, close quote marks, use semicolons at end of line
  • Trickier logical errors: check variables, draw a diagram of the process, "pseudocode" it, simplify, process of elimination
